{% capture x %} ### Asset Repository The assets available from the Asset Portal are listed in the central asset repository located here: [https://github.com/defold/awesome-defold](https://github.com/defold/awesome-defold) You can submit a new asset in three easy steps: 1) Fork the asset repository: ![For repository](/images/submit_asset/fork.png) 2) Add your asset to the bottom of the **assets.json** file. Example: ```json "library_url": "https://github.com/defold/defold-examples/releases/download/0.2.0/magic-link.zip", "description": "Magic Link is a simple game where the goal is to connect a set of magic spheres.", "author": "Defold", "images": { "thumb": "/images/assets/magiclink-thumb.jpg", "hero": "/images/assets/magiclink-hero.jpg" } }, { "name": "My cool new asset", "tags": ["Extension", "Device control"], "platforms": ["iOS", "Android", "macOS", "Windows", "Linux", "HTML5"], "license": "CC0 1.0 Universal", "website_url": "https://www.foobar.com/", "project_url": "https://github.com/foo/bar", "library_url": "https://github.com/foo/bar/archive/master.zip", "description": "Put more foobar in your project.", "author": "Mr Smith", "images": { "thumb": "/images/assets/foobar-thumb.jpg", "hero": "/images/assets/foobar-hero.jpg" } }, ] } ``` 3) Optionally add a thumbnail image and hero/banner image to the **images** folder. The thumbnail should be a maximum of 580x380 pixels and the the hero/banner image should be a maximum of 2400x666 pixels. 4) Submit a Pull Request back to the main asset repository. {% endcapture %}{{ x | markdownify }}
